Тема 1.4. Числа з плаваючою точкою та Математика

1. Типи даних з плаваючою точкою

У реальному житті не все можна порахувати цілими числами. Вага, температура, відстань, гроші — все це потребує дробів.

У Python був один тип float. У C++ їх три, і вони відрізняються точністю (кількістю знаків, які можуть запам'ятати).

Тип Пам'ять Точність (знаків) Рекомендація
float 4 байти ~6-7 знаків Для простої графіки, ігор (економія пам'яті)
double 8 байт ~15-16 знаків Стандарт! Використовуйте завжди за замовчуванням.
long double 10-16 байт 18+ знаків Для надточних наукових розрахунків
Чому double? Спробуйте зберегти число "пі" у float — після 7-го знаку почнеться "сміття" і помилки. double (подвійна точність) дає надійніший результат.

2. Форматування виводу (iomanip)

За замовчуванням C++ виводить стільки знаків, скільки вважає за потрібне (зазвичай 6). Але для грошей нам треба рівно 2 знаки (копійки), а для фізики — можливо 10.

Щоб керувати цим, підключаємо бібліотеку: #include <iomanip>

Основні "заклинання" (маніпулятори):

  • fixed — закріпити кому (заборонити науковий формат типу 1.5e+02).
  • setprecision(n) — встановити кількість знаків після коми.
#include <iostream>
#include <iomanip> // Обов'язково!
using namespace std;

int main() {
    double pi = 3.1415926535;
    
    // Вивести 2 знаки після коми
    cout << fixed << setprecision(2) << pi << endl; // 3.14
    
    // Вивести 5 знаків
    cout << fixed << setprecision(5) << pi << endl; // 3.14159
    
    return 0;
}

3. Інкремент та Декремент

Програмісти ліниві, тому вони придумали короткі команди для додавання та віднімання одиниці.

Назва Команда Аналог Опис
Інкремент i++; i = i + 1; Збільшити на 1 (Level Up)
Декремент i--; i = i - 1; Зменшити на 1

Примітка: Існує префіксна (++i) та постфіксна (i++) форми. Поки що використовуйте i++ як окрему команду.

4. Пропорції, Відсотки та Прогресії

4.1. Відсотки (Головна пастка)

У темі 1.2 ми вчили, що int / int дає ціле число. Це критично для відсотків!

Задача: Знайти 50% від числа 90.
Формула: (50 / 100) * 90

Помилка: 50 / 100 дасть 0 (бо це цілі числа). Вся відповідь буде 0.

Рішення: Писати 50.0 / 100.0 або використовувати змінні типу double.

4.2. Арифметична прогресія

Кожне наступне число більше за попереднє на d (крок). Наприклад: 2, 5, 8, 11...

[Image of C++ compilation process diagram]
n-й член: \( a_n = a_1 + d(n-1) \)
Сума n членів: \( S_n = \frac{(a_1 + a_n) \cdot n}{2} \)

4.3. Геометрична прогресія

Кожне наступне число множиться на q (знаменник). Наприклад: 3, 6, 12, 24... (росте дуже швидко, як вірус або популяція бактерій).

n-й член: \( b_n = b_1 \cdot q^{n-1} \)

Для степеня використовуємо функцію pow(q, n-1) з бібліотеки <cmath>.
