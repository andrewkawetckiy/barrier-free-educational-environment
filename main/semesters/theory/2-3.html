Тема 2.3. Компактні умови та Switch

1. Тернарний оператор (Скорочений if)

Іноді писати повний if-else на 5 рядків заради однієї простої дії — це занадто довго. Програмісти придумали "однорядковий if".

Його називають тернарним, бо він складається з трьох частин.

( Умова ) ? Дія_ТАК : Дія_НІ;

Порівняння з Python:

Python C++
res = "Good" if grade >= 60 else "Bad"
(Слова if та else всередині) 		res = (grade >= 60) ? "Good" : "Bad";
(Символи ? та :)

Приклад: Модуль числа (absolute value)

int x = -5;
// Якщо x менше 0, то беремо -x, інакше беремо x
int abs_x = (x < 0) ? -x : x; 
cout << abs_x; // Виведе 5

2. Оператор вибору (switch-case)

Уявіть кавовий автомат. Ви натискаєте кнопку "1" — отримуєте еспресо, "2" — лате, "3" — чай. Писати це через купу if (button == 1) ... else if (button == 2)... незручно.

Для перевірки конкретних значень (цілих чисел або символів) використовують switch.

Синтаксис:

switch (змінна) {
    case значення_1:
        // команди...
        break; // ВИХІД!
    case значення_2:
        // команди...
        break;
    default:
        // якщо нічого не підійшло (аналог else)
        break;
}

⚠️ УВАГА: Пастка з BREAK

У Python (в нових версіях match/case) вихід відбувається автоматично.
У C++ треба писати break вручну!

Якщо забути break, програма "провалиться" далі і виконає команди наступного case, навіть якщо він не підходить. Це називається fall-through.

Приклад: Меню гри

int choice;
cin >> choice;

switch (choice) {
    case 1:
        cout << "Start Game";
        break; // Без цього виконається і case 2!
    case 2:
        cout << "Settings";
        break;
    default:
        cout << "Unknown command";
}

3. Корисний трюк: Групування кейсів

Оскільки без break програма йде далі, ми можемо це використати для групування.

Задача: Визначити, чи це будній день, чи вихідний.

switch (day) {
    case 1:
    case 2:
    case 3:
    case 4:
    case 5:
        cout << "Робочий день :(";
        break; // break один на всіх!
    case 6:
    case 7:
        cout << "Вихідний!";
        break;
}
